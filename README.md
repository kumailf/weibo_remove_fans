# 微博垃圾粉丝清理工具

这是一个 Windows 命令行工具，用来批量移除同时满足以下条件的微博粉丝：

- 来源显示为“兴趣推荐”；
- 页面显示“回粉”，即对方关注了你，但你没有回关。

程序会自动打开 Chrome、进入粉丝列表并切换到“关注时间倒序”。开始移除前，终端会列出本批候选人的微博名字、来源和回关状态。

## 一、准备环境

需要准备：

- Windows 10 或 Windows 11；
- Google Chrome 浏览器；
- Python 3.10 或更高版本。

### 安装 Python

1. 打开 [Python 官网](https://www.python.org/downloads/windows/)；
2. 下载并运行 Windows 安装程序；
3. 安装界面务必勾选 **Add Python to PATH**；
4. 点击 **Install Now**。

安装完成后打开 PowerShell 或 Git Bash，输入：

```bash
python --version
```

能看到 Python 版本号即表示安装成功。

## 二、下载项目

在 GitHub 页面点击 **Code → Download ZIP**，解压到一个方便找到的目录。

打开解压后的项目文件夹，在文件夹空白处右键，选择“在终端中打开”。

## 三、安装依赖

在终端运行：

```bash
python -m pip install -i https://pypi.tuna.tsinghua.edu.cn/simple -r requirements.txt
```

只需安装一次。程序会使用电脑上已有的 Chrome，无需安装 Playwright 自带浏览器。

## 四、首次登录微博

运行：

```bash
python main.py login
```

程序会打开一个专用 Chrome 窗口。请扫码登录微博，登录成功后窗口会自动关闭。

Chrome 会按照当前屏幕的可用尺寸最大化打开。

登录信息保存在本机 `.data` 目录中，不要把该目录发送给别人。

## 五、清理粉丝

不加 `--limit` 时，会扫描所有匹配的候选用户并全部移除：

```bash
python main.py clean --confirm
```

建议第一次先小批量验证，例如只清理 3 个：

```bash
python main.py clean --limit 3 --confirm
```

`--limit` 是本次最多移除的人数，可以按需修改，例如：

```bash
python main.py clean --limit 10 --confirm
```

运行流程：

1. 自动进入个人主页和粉丝列表；
2. 切换到“关注时间倒序”；
3. 找到匹配的候选人（有 `--limit` 时只找指定数量，否则找全部）；
4. 在终端打印候选人的微博名字、来源和回关状态；
5. 在当前页面依次移除。

首次扫描候选名单时，下拉加载后会等待约 3 秒，让微博有足够时间加载新的粉丝卡片。

候选扫描结束后，程序会按候选顺序处理。为适配微博的动态列表，每次定位候选都会从顶部逐屏向下查找，并设置明确的查找次数上限；不会直接跳到列表底部、无限滚动或补充新人。

请先从小数量开始，确认结果符合预期后再不加 `--limit` 做全量清理。运行中按 `Ctrl+C` 可以停止。

如果某一轮因为悬停菜单、动态列表或临时页面错误而提前停止，程序会自动重新进入粉丝页、重新扫描并继续累计。有 `--limit` 时例如目标为 100 个、第一轮完成 29 个，下一轮会扫描并继续剩余 71 个；不加 `--limit` 时下一轮会重新扫描全部剩余匹配候选。连续 3 轮都没有成功移除时才会退出，避免无限重试。

## 六、只扫描，不移除

如果只想查看候选名单：

```bash
python main.py scan
```

结果会保存在：

- `.data/candidates.csv`
- `.data/candidates.json`

## 常见问题

### 登录失效或出现验证码

重新运行：

```bash
python main.py login
```

手动完成登录或验证。程序不会绕过验证码。

### Chrome 无法启动

关闭其他由本工具打开的 Chrome 窗口后重试。同一时间不要运行多个清理命令。

### 页面一直滚动或出现错误

按 `Ctrl+C` 停止程序，不要反复运行。微博页面结构发生变化时，程序可能需要更新。

悬停菜单偶尔可能因为页面加载延迟而没有出现，程序会自动重试 3 次；仍失败时会停止本批任务，可稍后重新运行。

### 清除本地登录信息

关闭程序后删除项目中的 `.data` 文件夹即可。下次使用需要重新扫码登录。

## 隐私与安全

- 不要在代码中填写微博账号、密码或 Cookie；
- 不要上传或分享 `.data`、`whitelist.txt` 等本地文件；
- 建议小批量操作，避免触发微博频率限制；
- 自动化操作可能受到微博规则限制，使用者需自行判断并承担风险。
